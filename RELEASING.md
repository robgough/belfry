# Releasing Belfry (macOS)

The release is a notarized, stapled, universal (arm64 + x86_64) `Belfry.app`,
attached to a GitHub release, with a Sparkle appcast so existing installs
(≥ 0.3.0) update themselves.

## One-time setup (already done on hailmary)

These live in the login keychain and only need doing again on a new machine:

- **Developer ID Application certificate** — from the Apple Developer portal
  (team `5Z5EG95CQL`). If `security find-identity -v -p codesigning` lists
  more than one with the same name, pass the SHA-1 hash as `SIGN_IDENTITY`
  instead of the name (codesign refuses ambiguous names).
- **Notarization credentials** — an app-specific password from
  [account.apple.com](https://account.apple.com), stored as a notarytool
  profile:

  ```sh
  xcrun notarytool store-credentials belfry-notary \
      --apple-id <apple-id> --team-id 5Z5EG95CQL
  ```

- **Sparkle EdDSA key** — keychain item "Private key for signing Sparkle
  updates". Generated by Sparkle's `generate_keys`
  (`.build/sparkle-tools/bin/generate_keys`); the matching public key is
  hard-coded as `SUPublicEDKey` in `scripts/make_app.sh`. Losing this key
  means shipped apps reject future updates — keep it backed up.

## Cutting a release

1. **Update `CHANGELOG.md`** — add a `## [X.Y.Z] — date` section at the top.

2. **Commit first.** The build number is `git rev-list --count HEAD`, so the
   released binary should be built from the commit you're about to tag.

3. **Build, notarize, staple, regenerate the appcast** (a few minutes; the
   wait is Apple's notary service):

   ```sh
   SIGN_IDENTITY=<cert-sha1-or-name> ./scripts/release.sh X.Y.Z
   ```

   This produces `Belfry-X.Y.Z.zip` (gitignored) and rewrites
   `docs/appcast.xml` with an EdDSA-signed entry pointing at the GitHub
   release asset URL. It ends with a Gatekeeper assessment that must say
   `accepted / source=Notarized Developer ID`.

4. **Tag, release, publish the appcast:**

   ```sh
   git add docs/appcast.xml && git commit -m "Appcast for X.Y.Z"
   git tag -a vX.Y.Z -m "Belfry X.Y.Z"
   git push origin main vX.Y.Z
   gh release create vX.Y.Z Belfry-X.Y.Z.zip --title "Belfry X.Y.Z" \
       --notes "<changelog section for X.Y.Z>"
   ```

   Order matters: the appcast's enclosure URL points at the release asset,
   and Sparkle clients read the appcast from
   `https://belfry.robgough.net/appcast.xml` — which is `docs/appcast.xml`
   published by GitHub Pages, so it goes live on push (~a minute for the
   Pages build).

5. **Verify.** Download the asset fresh and check Gatekeeper + the feed:

   ```sh
   curl -sLO https://github.com/robgough/belfry/releases/latest/download/Belfry-X.Y.Z.zip
   ditto -x -k Belfry-X.Y.Z.zip /tmp/belfry-verify
   spctl --assess --type execute -vv /tmp/belfry-verify/Belfry.app   # accepted, Notarized Developer ID
   curl -s https://belfry.robgough.net/appcast.xml | grep shortVersionString
   ```

   Belt-and-braces: launch the new app and hit "Check for Updates…" — it
   should say "You're up to date!".

## Gotchas

- **Stale build products.** The Swift build backend has been seen reusing
  stale binaries while reporting `Build complete` (tell-tale warning:
  `missing creator for mutated node`). If a change mysteriously doesn't take
  effect, check the binary really contains it
  (`strings Belfry.app/Contents/MacOS/Belfry | grep <new-literal>`) before
  debugging anything else.
- **Metal toolchain error** (`unable to spawn …/metal`): the Metal cryptex
  remounted under a new path and the build description cached the old one —
  `rm -rf .build/out/Intermediates.noindex/XCBuildData` and rebuild.
- **`release.sh --skip-notarize`** builds and signs without notarizing, for
  testing the pipeline itself.
- Sparkle tools are fetched automatically to `.build/sparkle-tools/` when
  missing; pin bumps happen in `scripts/release.sh` (tarball URL) and
  `Package.swift` together.
